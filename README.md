# Установка Ansible Playbook

В этой инструкции пошагово описан процесс подготовки чистого сервера, настройки доступов через GitHub и запуска плейбуков Ansible.

---

## Шаг 1: Подготовка чистого Debian 13

Зайдите на свой сервер по SSH под пользователем `root`. Сначала нужно обновить систему и установить базовые пакеты, которые понадобятся для работы.

```bash
apt update && apt upgrade -y
apt install git python3-venv sshpass nano curl -y
```

## Шаг 2: Создание ключа и привязка к GitHub

Этот шаг обязателен до настройки Ansible, потому что скрипт будет искать ваш публичный ключ в открытом доступе на GitHub, чтобы создать для вас безопасного пользователя на сервере.

**1. Генерируем ключ на сервере:**
```bash
ssh-keygen -t ed25519 -C "mail@example.com"
```
> **Внимание:** на все вопросы системы просто нажимай Enter. Нам нужен ключ со стандартным путем и без пароля (passphrase).

**2. Выводим ключ на экран:**
```bash
cat ~/.ssh/id_ed25519.pub
```
*Скопируйте весь выведенный текст (от `ssh-ed25519` до вашей почты).*

**3. Добавляем ключ в GitHub:**
1. Откройте браузер и зайдите в свой аккаунт на GitHub.
2. Перейдите в **Settings** (Настройки) -> **SSH and GPG keys**.
3. Нажмите зеленую кнопку **New SSH key**.
4. В поле **Title** напишите имя сервера (например, *Debian Remna*).
5. В поле **Key type** оставьте *Authentication Key*.
6. В большое поле **Key** вставьте скопированный ключ и нажмите **Add SSH key**.

**4. Проверяем доступность ключа:** Перейдите в браузере по ссылке `https://github.com/ТВОЙ_ЛОГИН.keys` (заменив `ТВОЙ_ЛОГИН` на свой реальный никнейм в GitHub). Если там появился текст вашего ключа — всё сделано идеально, можно переходить к Ansible.

## Шаг 3: Скачивание проекта и установка Ansible

Теперь скачиваем сам плейбук и настраиваем для него изолированное окружение (в Debian 13 это необходимо из-за системных ограничений Python).

```bash
# Клонируем репозиторий
git clone [https://github.com/sendmiche/ansible-playbook.git](https://github.com/sendmiche/ansible-playbook.git)

# Переходим в папку проекта
cd remna-playbook

# Создаем виртуальное окружение
python3 -m venv .env

# Активируем его (в начале строки терминала появится надпись .env)
source .env/bin/activate

# Устанавливаем Ansible и необходимые зависимости
pip install -r requirements.txt
ansible-galaxy install -r requirements.yml
```

## Шаг 4: Редактирование файлов конфигурации

Теперь нужно указать Ansible, где находится сервер и какой у вас логин на GitHub.

**1. Настройка инвентаря (IP сервера):**
```bash
cp inventory/inventory.yml.example inventory/inventory.yml
nano inventory/inventory.yml
```
* В открывшемся редакторе найдите строку с IP-адресом (обычно там `0.0.0.0` или пример IP) и замените на реальный IP-адрес вашего сервера.
* Сохраните файл: `Ctrl+O` -> `Enter` -> `Ctrl+X`.

**2. Настройка переменных (Ваш логин GitHub):**
```bash
cp group_vars/all.yml.example group_vars/all.yml
nano group_vars/all.yml
```
* Найдите переменную, отвечающую за имя пользователя, и впишите туда свой точный логин на GitHub (тот самый, по которому доступен ваш публичный ключ из Шага 2).
* Если в файле есть другие обязательные настройки (домены, почта для SSL, пароли для баз данных) — заполните их.
* Сохраните файл: `Ctrl+O` -> `Enter` -> `Ctrl+X`.

## Шаг 5: Запуск подготовки сервера (Создание пользователя)

Этот скрипт возьмет ваш логин из `all.yml`, скачает ключ из GitHub и создаст на сервере пользователя с таким же именем и настроенным доступом.

```bash
ansible-playbook prepare-playbook.yml -i inventory/inventory.yml -u root -k
```
* **Первый запрос пароля:** Введите пароль от пользователя `root` (серверный пароль).
* **Второй запрос пароля:** Придумайте и введите пароль для нового пользователя, которого сейчас создает Ansible (запомните его, он понадобится на следующем шаге).

## Шаг 6: Запуск установки проекта

Сервер подготовлен, права настроены. Запускаем основной процесс развертывания проекта. 
> Обратите внимание, что флаг сменился на заглавную `-K`.

```bash
ansible-playbook playbook.yml -i inventory/inventory.yml -K
```
* **Запрос пароля (BECOME password):** Введите пароль, который вы только что придумали для нового пользователя на Шаге 5.
